Project Documentation

Design choices, metadata model, and technical rationale for the Belvedere Virtual Exhibition.

1. Project Overview & Goals

This website is a MMMM (Multimedia Museum Meta Mirror) project created for the Information Modeling and Web Technologies course at the University of Bologna (A.Y. 2024–25, prof. Fabio Vitali). It serves as a virtual companion application for an imagined exhibition of Austrian art history at the Belvedere Museum, Vienna.

The exhibition presents 18 real artworks spanning from the late Baroque (c. 1770) to early Modernism (c. 1920), arranged in six thematic rooms and navigable through two distinct narrative paths. The system is designed to serve three user profiles:

Alice A middle-school teacher planning a 2-hour class trip. She needs quick, engaging descriptions and clear room structure. Served by the Short / Introductory / Young description mode and the Exhibition Map.
Bruno A busy professional with 45 minutes. He needs efficient, intelligent descriptions without excessive jargon. Served by the Medium / Average / Adult mode and the Timeline narrative for chronological orientation.
Carla A PhD student evaluating whether a physical visit adds value. She needs scholarly context. Served by the Long / Advanced / Scholar mode and the Emotion & Expression narrative for thematic depth.

2. Data Model (items.json)

All artwork data is stored in a single items.json file with three top-level keys: rooms, items, and narratives. This flat JSON architecture was chosen deliberately over a relational database to:

  • Enable purely client-side operation without a server backend
  • Allow easy inspection and editing by non-developers
  • Reduce network overhead (single fetch on page load)

2.1 Item Metadata Fields

Each item follows a Dublin Core-inspired schema, extended for museum use:

Why Dublin Core? Dublin Core is the most widely adopted metadata standard in digital cultural heritage (used by Europeana, the Getty, and the Belvedere's own online catalogue). Its 15 core elements map cleanly to painting metadata while remaining understandable to non-specialists. We use a simplified subset: title, creator, date, type, format, identifier (inventory number), and a custom room field for spatial placement.

2.2 Description Grid

Each artwork carries up to 5 descriptions keyed by a compound string (length_competence_tone) mapping to the grid specified in the project brief:

Length short (2–3 lines), medium (10–15 lines), long (scholarly depth)
Competence intro (no prior knowledge), avg (general interest), adv (specialist)
Tone adult (neutral professional), young (accessible, playful)

A graceful fallback chain in item.js tries the exact key first, then progressively relaxes constraints (e.g., medium_avg_adult → medium_intro_adult → medium), ensuring no description box is ever empty.

2.3 Narratives & Links

The narratives key stores three sub-keys: an ordered timeline array, an emotion_expression array, a links map (precomputed prev/next for O(1) navigation), and an index map (which narratives each item appears in). Precomputing links avoids array traversal on every navigation click.

3. Typographic Themes

The project implements two themes, each inspired by a distinct historical aesthetic appropriate to the exhibition's subject matter.

3.1 Classic Theme - XIX Century Academic

The Classic theme references the visual culture of 19th-century Viennese museum catalogues and the Biedermeier period (1815–1848). Every choice is grounded in period precedent:

Background (#f4ede0): This warm parchment tone matches the paper stock used in Viennese cultural periodicals such as Allgemeine Theaterzeitung (1840s). Pure white would be anachronistic - 19th-century printing could not achieve optical white.
Fonts - Playfair Display + Lora: Playfair Display evokes the high-contrast display serifs (Didot, Bodoni) found in exhibition catalogues of the Kunsthistorisches Museum's founding era (1891). Lora provides a readable body serif that references letterpress printing without literal historical simulation.
Accent colour (#b5892a): A warm ochre-gold chosen to reference the gilded frames of academic salon paintings of this period. It also functions as the high-contrast signal colour required for accessibility (contrast ratio ≥ 4.5:1 against the background).
Decorative border: The repeating ochre-stripe top border references the ornamental printed rules used in Austrian state publications and museum catalogues of the 1860s–1890s (Ringstrasse era). It signals cultural authority without being obtrusive.
Image framing (padding + box-shadow offset): Images in the item view are wrapped with a 12px padding and a flat shadow offset, mimicking the look of a mounted print on a museum mat board. This is grounded in the display conventions of the period and creates clear visual hierarchy between image and text.

3.2 Modern Theme - Vienna Secession / Jugendstil 1900

The Modern theme is inspired by the Vienna Secession (founded 1897), specifically the graphic language of Ver Sacrum magazine (1898–1903) and Koloman Moser's ornamental work:

Dark background (#0e0e0e): The near-black references the Secession building's dark zinc roof and the chiaroscuro of Klimt's gold-ground paintings. Dark backgrounds also mirror the low-lit atmosphere of the Secession's original exhibition rooms, designed to focus attention on the luminosity of the works.
Gold accent (#d4a843): Klimt's literal use of gold leaf in The Kiss (1907–08) and Judith I (1901) makes gold the definitive Secession colour. The specific value matches the warm yellow-gold (as opposed to cool platinum) of gold leaf applied over dark grounds.
Fonts - Cormorant Garamond + Inter: Cormorant's extreme contrast ratio (thin hairlines vs. thick strokes) echoes the Otto Wagner lettering characteristic of Jugendstil architecture. Inter for navigation UI is a deliberate anachronism signalling that the digital interface does not pretend to be a period artefact.
Geometric border pattern: The repeating gold dash–gap pattern at the document top references Koloman Moser's geometric surface ornament designs for Ver Sacrum covers (1898). It establishes visual rhythm without figurative decoration.
Typography scale - uppercase small caps for labels: The Secession and its contemporaries (e.g., Deutsche Werkstätten) used structured uppercase lettering for museum labels. The 0.15em letter-spacing on category labels is calibrated to produce the spaced-out look of lithographic uppercase display type of the period.

4. Interactive Features

4.1 Theme Switching

Theme switching is handled by dynamically replacing a <link> element (id: theme-style) via theme.js. The chosen theme is persisted in localStorage so it survives navigation between pages. A body class (light / dark) is applied in parallel for CSS variable inheritance in timeline.css.

Why a separate CSS file per theme rather than CSS custom properties + a class toggle? Separate files allow complete typographic overrides (font-family, line-height, font-size) that cannot be expressed through CSS custom properties alone. This approach also makes each theme's code independently legible and maintainable.

4.2 Exhibition Map

The map uses a CSS grid to represent room adjacency in a schematic floor-plan layout. Clicking a room triggers showRoom() in map.js, which fetches items from the JSON and populates a slide-in panel. The panel uses CSS transitions (opacity + translateY) rather than JavaScript animation for performance and accessibility.

4.3 Artwork Description Controls

Three dropdowns (Length, Competence, Tone) map to the three axes of the description grid. Changes trigger loadDescription() which applies the fallback chain described in §2.2. All dropdowns are standard <select> elements (no custom components) to preserve native keyboard and screen-reader accessibility.

4.4 Narrative Navigation

Each narrative defines a complete ordered list of artwork IDs. Navigation between artworks is implemented by passing ?id=X&n=narrativeName query parameters in the URL - enabling bookmarking and direct linking to any artwork within any narrative.

4.5 Timeline (timeline.js)

The timeline reads the narratives.timeline array for its ordered sequence. An IntersectionObserver triggers a CSS reveal animation (.hidden → .in-view) as items scroll into view, replacing a JavaScript-heavy approach with a pure CSS transition. This reduces repaints significantly on long timelines.

5. Responsive Design

The layout uses three breakpoints: 1100px (wide), 900px (medium, collapses item grid to single column), and 800px (mobile, hamburger navigation, single-column grids). The artwork marquee on the homepage uses a CSS animation: marquee-scroll with will-change: transform for GPU-composited scrolling, avoiding layout thrashing on mobile.

6. Accessibility

  • All navigation uses semantic <nav>, <main>, <header>, and <footer> landmarks.
  • Images carry descriptive alt text populated from item titles.
  • The mobile nav toggle uses aria-expanded and aria-controls attributes.
  • The room panel uses aria-hidden toggling.
  • Colour contrast in both themes meets WCAG AA (4.5:1 for body text).
  • All interactive elements are keyboard-navigable.
